## 6. Workflow — порядок виконання {#workflow}

### Принципи
```
1. Сервісна сторінка (/services/) ЗАВЖДИ перша → потім local pages
2. Один сервіс повністю (service page + hub + local 26 міст) = один спринт
3. Не починати наступний сервіс поки не завершено поточний
4. Industry pages — паралельно після Хвилі 1
5. Blog кластери — паралельно, по 2-3 статті на тиждень
```

## Правила та заборони

### Дозволено
```
✅ Generative approach: шаблон + city-specific data = local pages
✅ Anonymized case studies з реальними метриками
✅ Порівняльні сторінки на основі верифікованих фактів
✅ aggregateRating у schema ТІЛЬКИ якщо є visible reviews on-page
✅ Окремий service slug для кожної послуги в /local/
```

### Заборонено
```
❌ Identичний контент між містами (тільки заміна назви)
❌ Fake reviews або inflated metrics у schema
❌ "We guarantee #1 on Google" або similar
❌ Копіювати контент конкурентів
❌ Загальні фрази без доказів ("world-class", "best in class")
❌ aggregateRating без видимого review content на сторінці
❌ Comparison pages з неперевіреними tverdzhenniami про конкурентів
```
